System Architecture Overview

A comprehensive layered architecture for the Public Grievance Redressal System, ensuring modularity, scalability, and security.

5 Layers

Modular design for maintainability

Java Stack

Backend and desktop applications

Multi-Platform

Layered Architecture Diagram

User Layer

Citizen - JavaFX Desktop App

Client-side application for citizens to submit and track complaints.

Citizen - Android App

Mobile application with camera integration and location services.

Frontend Application

Complaint Submission Form

Captures complaint details, file attachments, and user information.

Tracking Dashboard

Real-time status updates and a timeline of complaint progress.

Charts & Analytics (JFreeChart)

Visual representation of complaint trends and resolution metrics.

Backend Layer (Java)

Authentication & Authorization

Handles user login, session management, and access control.

Complaint Workflow Management

Manages complaint lifecycle, status changes, and assignments.

Escalation Scheduler (Quartz)

Automates the escalation of unresolved complaints based on SLAs.

Notification System (JavaMail API)

Sends email and SMS alerts to users and authorities.

Data Layer

Relational Database (MySQL / PostgreSQL)

Stores structured data for citizens, complaints, authorities, and escalation history.

Evidence File Storage (BLOBs)

Manages binary large objects (BLOBs) with integrity hashing for documents and images.

Admin / Authority Interface

Dashboard (JavaFX / JSP)

Administrative panel for monitoring, assigning complaints, and system management.

Status Update Module

Enables authorities to update complaint status and add notes.

Reporting Module (JasperReports)

Generates custom and scheduled reports from the system data.
